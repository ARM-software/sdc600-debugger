19 ## External COM Port Driver @ref ext_com_port_driver.h
20 
21 SDC-600 External COM Port driver runs in the debugger platform and it owns and
22 manages the SDC-600 External COM port at the debugged system.
23 The driver serves a Secure Debug Manager component in the debugger and acts per its needs.
24 The driver does not transmit or receive by itself. The driver provides some APIs to the
25 Secure Debug Manager.

27 ## CSAPBCOM (I/O driver) @ref csapbcom.h
28 
29 The CSAPBCOM interface an implementation of the I/O driver layer to allow reading/writing of generic
30 data to an ARM CoreSight&tm; COM-AP or APBCOM device, via Arm DSTREAM family debug probes.
31 
32 Further to data reads/writes, system reset functionality via the debugged systems DAP is provided. The
33 debugger therefore does not have to provide system reset callbacks for the External COM Port Driver if
34 it is known that the debugged system implements secure debug certificate processing in ROM.
