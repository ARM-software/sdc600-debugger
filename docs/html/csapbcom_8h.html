typedef enum CSAPBCOMReturnCode CSAPBCOMReturnCode

Return codes from CSAPBCOM_* functionality.

These return codes delineate various responses from the function calls featured in this API. All functions should return CSAPBCOM_SUCCESS if the debug transaction is successful, otherwise the subsequent error codes describe the error encountered.

enum CSAPBCOMReturnCode

Return codes from CSAPBCOM_* functionality.

These return codes delineate various responses from the function calls featured in this API. All functions should return CSAPBCOM_SUCCESS if the debug transaction is successful, otherwise the subsequent error codes describe the error encountered.

Enumerator
CSAPBCOM_SUCCESS 

Success, no error

CSAPBCOM_BADARG 

The client passed a bad argument to an CSAPBCOM function - typically a NULL pointer, a zero-length buffer or a zero device ID

CSAPBCOM_INVALID_HANDLE 

The session handle passed to the CSAPBCOM function was invalid

CSAPBCOM_FAILED 

Non-specific failure

CSAPBCOM_TOO_MANY_CONNECTIONS 

All available connections are used. Only returned by CSAPBCOM_Open

CSAPBCOM_BUFFER_OVERFLOW 

Provided buffer is too small

CSAPBCOM_INTERNAL_ERROR 

An unspecified internal error occurred

CSAPBCOM_PARSE_FAILED 

CSAPBCOM_Open failed to parse the SDF file specified in CSAPBCOMConnectionDescription

CSAPBCOM_REG_ACCESS 

Register access failed

CSAPBCOM_TIMEOUT 

Operation timed out

CSAPBCOM_RW_FAIL 

Operation failed to access memory or a register for reasons not covered by any other reason

CSAPBCOM_DEV_UNKNOWN 

A command has been sent to a device that is not listed in the SDF file

CSAPBCOM_DEV_IN_USE 

Returned when the client tried to make an active connection to a device that already has an active connection

CSAPBCOM_NO_CONN 

No connection has been made to the specified vehicle. Ensure CSAPBCOM_Connect has been called successfully and the client has not disconnected.

CSAPBCOM_COMMS 

General error returned when there is a problem with the communications channel

CSAPBCOM_DEV_BUSY 

The vehicle has been left in a busy state by a previous call or by another client

CSAPBCOM_NO_INIT 

No connection has been made to the specified device. Ensure CSAPBCOM_Open has been called successfully

CSAPBCOM_LOST_CONN 

The connection to the remote vehicle or device has been lost

CSAPBCOM_NO_VCC 

The device is not powered or has been disconnected

CSAPBCOM_NO_RESPONSE 

The requested operation failed because it timed out waiting for a response from the device

CSAPBCOM_OUT_OF_MEM 

Unable to allocate sufficient memory to complete the requested operation

CSAPBCOM_WRONG_DEV 

Device is not a SDC-600 COM port device. Either COM-AP or APBCOM

CSAPBCOM_NODEBUGPOWER 

The debug system on the target is not powered

CSAPBCOM_UNKNOWN 

An unknown error was encountered

CSAPBCOM_NO_CONFIG_FILE 

The SDF file supplied to in CSAPBCOMConnectionDescription to CSAPBCOM_Open cannot be found

CSAPBCOM_UNKNOWN_CONFIG 

An unknown SDF file error was encountered

CSAPBCOMReturnCode CSAPBCOM_GetInterfaceVersion ( char *  versionDetails,
size_t  versionDetailsLength 
)

Retrieve interface implementation version details.

Calling this function allows clients of the library to retrieve specific implementation details before opening a connection to the library.

This facilitates multiple code-paths depending on the returned version and the context it implied. It is the responsibility of clients of this library to interpret the version details correctly and act accordingly.

Parameters
[out]versionDetailsClient supplied buffer to retrive an implementation defined version string. The returned string will be NUL terminated.
[in]versionDetailsLengthThe size in bytes of the supplied versionDetails buffer. Note that a NUL terminating character will always be appended so clients must account for this extra size. Should an overflow be encountered the string will be truncated and an appropriate error message will be returned.

◆ CSAPBCOM_Open()

CSAPBCOMReturnCode CSAPBCOM_Open ( CSAPBCOMHandle *  outHandle,
const CSAPBCOMConnectionDescription *  topology 
)

Open a connection to the CSAPBCOM library and connect to the debug vehicle.

This is the initial function all clients must call when interacting with the library. A unique handle will be returned and must be stored for subsequent calls to the API.

Internally, this function connects to the debug vehicle using using an address and protocol specified in the topology argument.

If the debug vehicle cannot be connected to, or an error occurs during the process, a relevant error code will be returned.

Parameters
[out]outHandlePointer to a handle. This function returns a unique handle which identifies the current session and must be used in each of the following calls to the library.
[in]topologyImplementation-specific data containing details regarding the connection to the debug vehicle, and the target.

CSAPBCOMReturnCode CSAPBCOM_WriteData ( CSAPBCOMHandle  handle,
int  block,
size_t  numBytes,
const unsigned char *  inData 
)

Write generic data to the APBCOM TxEngine.

Data is written according to the TxEngine width so there is no need to pad inData with NUL bytes.

Recommended for byte flags and small messages.

Parameters
[in]handleHandle previously retrieved by call to CSAPBCOM_Open()
[in]blockIf zero no consideration is taken for the available space in the TxEngine, it is therefore up to the caller to check available space using CSAPBCOM_GetStatus. If data is written without free space being available, an overflow will occur. If non-zero writes are blocked until there is sufficent space in the TxEngine.
[in]numBytesThe number of bytes to write
[in]inDataA client supplied buffer to write the requested data to the TxEngine
